Project Case Study

AI Résumé Analyzer

AI Résumé Analyzer is a generative AI web application that helps candidates assess how well their résumé aligns with a target job description. Users upload a résumé in PDF or DOCX format, paste a job posting, and receive structured feedback on matched skills, gaps, and explanatory context.

App type LLM-powered web app
Backend Flask + Python
Documents PDF / DOCX parsing
Platform Gateway-ready multi-service architecture

Overview

This application is part of jr-portfolio-projects, a multi-service AI demo platform designed to host independent applications behind a single gateway. AI Résumé Analyzer focuses on a common job-seeker pain point: understanding not just whether a résumé matches a role, but why.

Rather than relying on simple keyword matching, the application combines document parsing, prompt orchestration, and LLM-based reasoning to generate a readable analysis of strengths, missing qualifications, and likely alignment with the role.

Problem

Many résumé review tools are either too shallow or too opaque. Keyword-only tools often miss contextual fit, while black-box AI outputs can be hard to trust. I wanted to build a version that provides a clearer, more explainable assessment while also fitting into a reusable multi-application architecture.

From an engineering perspective, the project also needed to support multiple file formats, structured prompting, responsive UI states, and a deployment model that could later sit behind a shared gateway in a Lightsail-based environment.

Solution

Multi-format résumé ingestion

Supports PDF and DOCX uploads with format-specific parsing and normalized text extraction before analysis.

Structured prompt orchestration

Builds a consistent analysis prompt from résumé content and job requirements to improve response quality and readability.

Explainable AI output

Returns matched qualifications, missing skills, and contextual reasoning instead of a generic relevance score.

Gateway-ready platform design

Organized as a service within a broader multi-app platform, enabling routing through a shared FastAPI gateway.

Technical Architecture

Application Flow

Platform Positioning Inside jr-portfolio-projects

Frontend

  • HTML, CSS, JavaScript
  • Responsive upload flow and animated results presentation
  • Tabbed experience for project overview and live demo

Backend

  • Flask application with résumé parsing routes
  • PyPDF2 for PDF parsing and python-docx for DOCX extraction
  • Input preprocessing, prompt assembly, and response formatting

AI Layer

  • OpenAI models for skill extraction and semantic comparison
  • Prompt design focused on explainable, user-readable output
  • Structured result formatting to highlight fit and gaps

Deployment Pattern

  • Runs standalone or behind a shared gateway
  • Gateway uses host-based routing and proxying for service isolation
  • Designed for Docker and AWS Lightsail Container Service

Results & Impact

  • Built a working AI product around a practical user workflow: résumé-to-role evaluation.
  • Demonstrated end-to-end ownership across UI, backend logic, LLM integration, and routing architecture.
  • Validated a reusable gateway pattern for hosting multiple AI demo applications behind one public entry point.
  • Created a portfolio-ready application that shows both product thinking and hands-on implementation.

Key Takeaways

  • Effective AI applications depend on more than model access; parsing, normalization, and prompt design matter just as much.
  • Gateway-based multi-service design makes it easier to add new applications without tightly coupling them together.
  • Explainable output is especially important in decision-support tools where users need to understand the reasoning, not just the result.
